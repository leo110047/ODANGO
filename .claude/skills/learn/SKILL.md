# 學習與記錄機制

## 自動觸發條件

當發生以下情況時，**不需要用戶提醒**，必須**立即主動**更新對應的 skill：

1. **操作失敗** - CI 失敗、建置錯誤、更新失敗等
2. **用戶糾正** - 用戶指出做錯了什麼
3. **發現新的注意事項** - 過程中發現的坑或限制
4. **解決了問題** - 找到問題原因和解決方案後

## 記錄流程

### 1. 立即識別問題類型

判斷這個錯誤屬於哪個領域：
- 發布流程 → `.claude/skills/release/SKILL.md`
- Git 操作 → `.claude/skills/git/SKILL.md`（如不存在則建立）
- 建置問題 → `.claude/skills/build/SKILL.md`（如不存在則建立）
- 專案特定 → `CLAUDE.md` 或建立新 skill

### 2. 立即更新對應的 skill 文件

**不要等用戶提醒！** 直接修改對應的 skill 文件，加入：
- 明確的警告標記（⚠️）
- 錯誤的描述和錯誤訊息
- 正確的做法
- 為什麼會出錯

### 3. 告知用戶已記錄

更新後主動告知用戶：
> 已將此教訓記錄到 `/release` skill 中。

## 重要

- **不要把教訓寫在這個 `/learn` 文件裡！** 這裡只是指導如何記錄
- **不要等用戶說「記錄下來」才記錄！** 問題解決後立即記錄
- **每次犯錯或被糾正後，第一件事就是更新 skill**
